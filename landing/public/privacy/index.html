1. What data we collect

React Perf Profiler operates entirely on your local machine. The extension does not have a backend, does not create accounts, and does not transmit any data to any server operated by this project.

The only data processed by the extension is:

  • React component render data captured from the page you are actively profiling
  • Performance profiles you choose to export or sync (see sections 4–6 below)
  • Extension preferences you configure (stored locally in chrome.storage.local)

2. What we do NOT collect

  • Your name, email address, or any identifying information
  • The contents of pages you profile (DOM text, URLs, form data)
  • Browsing history or navigation patterns
  • Analytics, telemetry, or usage statistics
  • Crash reports (errors are logged to the browser console only)
  • Any data from pages where you are not actively using the profiler panel

3. Local data storage

The extension stores the following data locally using browser storage APIs:

  • Profile sessions — Stored in IndexedDB for session recovery. Cleared when you discard a session or uninstall the extension.
  • Extension preferences — Stored in chrome.storage.local. Includes panel layout, theme, and feature toggles.
  • API keys — If you configure the AI Suggestions panel, your API key is stored in chrome.storage.local. It is never transmitted except directly to the AI provider of your choice when you request a suggestion.

None of this data is synced to any server operated by this project. It stays on your machine.

4. AI Suggestions Panel

The AI Suggestions panel is an opt-in feature that requires you to supply your own API key. Supported providers: OpenAI, Anthropic, and local Ollama instances.

When you request a suggestion:

  • Component name, render count, wasted render count, and detected props are sent to the provider you configured, using your API key.
  • No page content, URLs, or personally identifying information is included in the prompt.
  • Requests go directly from the extension to the provider — this project has no intermediate server.
  • Using a local Ollama instance means no data leaves your machine at all.

Refer to your chosen provider's privacy policy for how they handle API requests: OpenAI | Anthropic.

5. Cloud sync (optional)

The cloud sync feature lets you export profiles to S3, Dropbox, or Google Drive. This feature is disabled by default and only activates when you enter your own cloud storage credentials.

  • Credentials are stored locally in chrome.storage.local.
  • Profiles are sent directly to your storage provider using your credentials.
  • This project does not operate any storage infrastructure.

6. Team sessions (optional)

Live team sessions use WebRTC for peer-to-peer communication. A signaling relay server is used only to establish the WebRTC connection — it does not store profile data.

  • The relay passes WebRTC handshake messages (offers, answers, ICE candidates) between peers. Profile data flows directly peer-to-peer after the connection is established.
  • You can run your own relay server using the included scripts/signaling-server.js.
  • Session codes and join tokens expire after 30 minutes.

7. Browser permissions explained

The extension requests the following permissions:

  • devtools — Required to add the Perf Profiler tab to Chrome/Firefox DevTools and access the React DevTools hook.
  • storage — Required to store preferences and API keys locally in chrome.storage.local.
  • tabs — Required to associate profiling sessions with specific browser tabs and clean up data when a tab closes.
  • alarms — Required to keep the service worker alive during long recording sessions (Chrome MV3 only).
  • scripting — Required to inject the profiler bridge script into the page being profiled.

No permission grants access to your browsing history, bookmarks, passwords, or data from pages you are not actively profiling.

8. Third-party services

This project uses the following third-party services — none of which receive profiling data:

  • GitHub — Hosts the source code and issue tracker.
  • GitHub Pages — Hosts this landing page. GitHub may collect standard web server logs (IP address, timestamp, page requested). See GitHub's Privacy Statement.

9. Contact

If you have questions about this privacy policy, please open an issue on GitHub.

Because the extension collects no personal data, there is no data to access, correct, or delete.
